# Field-to-Industry Translation Guide

> **Purpose:** Help EIC users translate academic and campus experiences into professional language that resonates with hiring managers and passes ATS screening.
>
> **Who this is for:** Students and recent graduates from any major — especially those pivoting from academia into industry roles for the first time.
>
> **How to use with the Resume Builder prompt:** When the prompt asks you to describe an experience, look up your academic activity in the tables below. Use the industry phrasing as a starting point, then add specific metrics, tools, or outcomes to make it yours.

---

## Part 3: Usage Notes

### When to Translate Literally

Use the industry phrase as a **direct replacement** when:

- The academic activity and the industry task are functionally the same (e.g., "survey design" → "primary research" — you literally did the same work)
- The job description uses the industry term and your experience maps cleanly to it
- You need to pass ATS screening and the employer's posting uses the industry phrasing

**Example:**
> ❌ "Conducted a literature review on cognitive behavioral therapy outcomes"
> ✅ "Conducted competitive landscape analysis of behavioral health intervention models, synthesizing 30+ peer-reviewed sources into actionable recommendations"

### When to Add Context

Add a brief qualifier or bridge phrase when:

- The translation is a **stretch** — the activity is related but not identical to the industry task
- You're pivoting into a very different sector and need to show transferability
- The reader might not understand the academic context without a short explanation

**Bridge phrases to use:**
- "…applying the same methodology used in [industry term]"
- "…equivalent to [industry task] in a corporate setting"
- "…requiring the same skills as [industry activity]"

**Example:**
> ❌ "Graded undergraduate papers"
> ✅ "Evaluated 50+ student deliverables per week against standardized rubrics, providing structured written feedback — a process mirroring quality assurance review in professional settings"

### When NOT to Translate

- **Don't obscure genuine technical expertise.** If you published in a peer-reviewed journal, say so — that carries weight in research-adjacent roles.
- **Don't translate if the academic term is the industry term.** "Statistical analysis" is already professional language. "IRB compliance" is already understood in health tech.
- **Don't over-translate for academic positions.** If you're applying for a TA role or research assistantship, use academic terminology — that's the industry language.

### Combining This Guide with the Resume Builder Prompt

1. **Before starting the resume builder:** Scan the [Sample Job Descriptions](job-descriptions/sample-jds.md) for your target role. Note the ATS keywords.
2. **During the experience section:** For each bullet point, check this translation guide. Find your academic activity, grab the industry phrasing, and weave in the ATS keywords from step 1.
3. **After drafting:** Read each bullet point aloud. If it sounds like a course catalog, translate further. If it sounds like a job posting, you're on track.

> **Remember:** The goal isn't to hide your academic background — it's to help employers see how your skills already match what they need.
